## Q3. レスポンス形式の使い分け

以下の3つの使い分けを説明してください。

```ts
c.text('Hello')
c.json({ ok: true })
c.html(<View />)
```

A. シンプルにテキストかjsonかhtmlかっていう説明しかできないや

**△ 半分正解** — 「テキスト・JSON・HTML」という区別は合っています。補足すると、重要なのは **Content-Type ヘッダーが自動で変わる**点です：

| メソッド | Content-Type |
|---|---|
| `c.text()` | `text/plain` |
| `c.json()` | `application/json` |
| `c.html()` | `text/html` |

フロントエンドやブラウザは Content-Type を見てデータの扱い方を決めるので、この自動設定が地味に重要です。

## Q4. ステータスコード

```ts
app.post('/posts', (c) => c.text('Created', 201));
```

`201` を省略して `c.text('Created')` にすると、ステータスコードはいくつになりますか？

A. 201ではないのか？なんかステータスコードの意味の違いやったけど忘れたな。成功で200だっけ？

**△ 惜しい** — 省略すると `200` になります。

- `200 OK` → リクエスト成功（取得・更新など）
- `201 Created` → 新しいリソースが**作成**された

POST でリソース作成したときは `201` が正しい慣習です。今のコードは明示的に `201` を渡しているのでちゃんと正しいです。

## Q5. Context とは何か

ハンドラーの引数 `c` は何を表していますか？ `Request` オブジェクトと何が違いますか？

A. これはわからない。claudeflareのcなのかと思ってた
このcは何を意味するの？

**❌ 解説** — `c` は **Context（コンテキスト）** の略です。Hono が独自に定義したオブジェクトで、リクエストとレスポンスの両方を1つにまとめたものです。

```ts
c.req.param()   // リクエスト側
c.text()        // レスポンス側
c.header()      // レスポンスヘッダー操作
```

Web 標準の `Request` はリクエスト情報しか持ちませんが、`c` はレスポンスも作れる点が違います。

## Q6. JSX を使うために必要なこと

```tsx
const View = () => (
  <html>
    <body>
      <h1>Hello!</h1>
    </body>
  </html>
);
```

このファイルが `.tsx` になっている理由と、`tsconfig.json` に必要な設定は何ですか？

A. htmlをreturnするために変更した
tsconfig.jsonはいじってないのでわからないです

**△ 半分正解** — 「HTML を return するため」という理解は正しいです。より正確には **JSX 構文（`<html>...</html>` のような HTML っぽい書き方）を TypeScript 内で使うため**です。

`tsconfig.json` には以下の設定が必要で、このプロジェクトにはすでに設定済みです：

```json
"jsx": "react-jsx",
"jsxImportSource": "hono/jsx"
```

`jsxImportSource` が `hono/jsx` になっているのがポイントで、React ではなく **Hono の JSX ランタイム**を使うよう指定しています。この設定があるから `<View />` が動きます。
